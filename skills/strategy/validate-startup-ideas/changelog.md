# Changelog

All notable changes to the `validate-startup-ideas` skill will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

## [1.6.0] - 2026-02-19

### Changed

- Added intro note: apply frameworks as lenses to what the user brings, works at any stage of idea maturity
- Reorganized sections under find / evaluate / shape / test groupings
- Moved sugar-coated broccoli from under "Validation signals" to "Shaping the solution" where it belongs
- Renamed "Validation signals" → "Evaluating ideas" and "Rapid validation process" → "Testing early"

## [1.5.0] - 2026-01-17

### Added

- Adjacent users framework (Bangaly Kaba): solve for users just outside current base
- Tarpit detection signals (Dalton Caldwell): warning signs for systematically-failing ideas
- Sugar-coated broccoli principle (Ivan Zhao): hide ambitious vision in familiar form factors
- Force a choice, not a comparison (Mike Maples Jr): be the first banana, not a better apple
- Inflections and earned secrets framework (Mike Maples Jr): pattern-breaking ideas at intersection of external events and non-obvious insights
- Value-business balance and gradual PMF (Ivan Zhao): art project ↔ commodity spectrum
- Don't be afraid to reset with better abstractions (Ivan Zhao)

## [1.0.0] - 2025-01-01

### Added

- Initial skill with premise framework, validation signals, idea maze, and evolution philosophy
