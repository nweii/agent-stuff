# Durable learning

When the user is building understanding meant to outlast the session, some things are worth keeping so a later session starts where this one ended. Defer to how the user already keeps things rather than imposing a format. Remember to follow any existing applicable user preferences or skills for writing voice, unless otherwise instructed.

## Learning records

Save a short note of what the user has come to understand, written so a future session builds on it instead of re-teaching. Worth recording when:

- They demonstrated they can use a non-trivial concept correctly, not just sat through an explanation of it.
- They disclosed prior knowledge ("I already know X"); note the depth they claimed.
- A misconception got corrected; these predict where related topics will trip them up.

A record can be a sentence or two: what is now known, and why/how it changes what to teach next. When a later understanding replaces an earlier one, mark the old note as superseded rather than deleting it; the way understanding evolved is worth keeping.

## Glossary

The topic's terms, each compressed to a tight definition. Add a term only once the user can use it correctly, since writing the definition is part of the learning and a record that it landed. Be opinionated: when several words mean the same thing, pick one and list the rest as aliases to avoid. Keep definitions to a sentence or two, and use the glossary's own terms inside other definitions. Most useful for topics with their own vocabulary.

## Resources

The trusted sources teaching draws on, so explanations rest on solid material rather than guesses. Two kinds are worth tracking: sources to learn from (books, papers, documentation, recognized experts), and communities where the user can test their skills against other people (a forum, a class, a local group). Annotate each with what it covers and when to reach for it. Keep only high-trust entries; a handful of sharp ones beats a long shallow list.
